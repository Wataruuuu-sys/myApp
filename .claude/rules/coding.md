# コーディング基本ルール

- SOLIDの原則を意識する
- 関数（method, function）は30行を目安にする
- コードのネストは3以上には絶対しない、努力目標で2を目安にする
- 過剰なtry-catchは避ける。fallback的な処理は行わない、fallbackが不要になるように設計する
- コメントは意味のあるWHYのみをのこし、日本語で記載する
- typescriptならany、unknown, Record使用禁止
- 将来的に必要なコードは実装しない（YAGNI）
- 関数名、変数名、クラス名に明示的な名称は含めない
  - 取得ロジックにgetは使用しない。何を取得しようとしているかを簡潔に命名
  - セッター、ゲッターにset~やget~のプレフィックスは不要
  - 関数に~~function, methodなどのsuffixは不要
  - 変数名にvalueなど曖昧な定義は採用しない。
